# 调查问卷设计与评估方法论

> 知识来源：《调查问卷的设计与评估》(Improving Survey Questions: Design and Evaluation)  
> 作者：弗洛德·J·福勒(Floyd J. Fowler, Jr.)  
> 蒸馏时间：2026-06-23

---

## 一、核心概念：什么是好问题？

### 1.1 测量的本质
- **信度**：答案的一致性程度（当状态一致时，答案也一致）
- **效度**：答案与"真值"的一致程度
- **好问题的标准**：引出对目标对象可靠、有效测量答案的问题

### 1.2 好问题的五个基本特征

| 特征 | 含义 | 关键问题 |
|------|------|----------|
| **理解一致性** | 所有受访者用一致方式理解问题 | 受访者理解是否与研究者意图一致？ |
| **管理一致性** | 问题能用一致方式呈现 | 书面可读？访员可逐字念出？ |
| **答案类型一致性** | 明确沟通"什么是充分答案" | 受访者是否清楚期望的答案形式？ |
| **可回答性** | 受访者拥有回答所需信息 | 除非测量知识，否则不应问不知答案的问题 |
| **愿意回答** | 受访者愿意提供正确有效答案 | 答案是否会受到社会期望偏差影响？ |

## 二、事实资料收集的问题设计

### 2.1 问题设计的五大挑战

```
┌─────────────────────────────────────────────────────────────┐
│  1. 界定目标 → 明确所需信息类型                              │
│  2. 概念界定 → 确保所有受访者对关键术语理解一致              │
│  3. 知道答案 → 克服信息缺乏、记忆困难、时间定位困难         │
│  4. 能回答 → 受访者能用问题要求的术语回答                   │
│  5. 愿意回答 → 减少社会期望偏差影响                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 概念与术语界定策略

**核心原则**：未界定的术语会产生20%-30%的误差

**案例分析**：
- ❌ "你上周有几天吃过黄油？"（黄油vs人造黄油混淆）
- ✅ "你上周有几天吃过黄油（不包括人造黄油）？"

**界定方法**：
1. 提供明确定义
2. 举例说明包含/排除范围
3. 使用多个问题逐步精确

### 2.3 记忆与回忆策略

**三类回忆障碍**：
| 障碍类型 | 解决策略 |
|----------|----------|
| 缺乏信息 | 不询问受访者不知道的问题 |
| 记忆模糊 | 使用联想提示、情境重构 |
| 时间定位困难 | 使用日历、生活事件锚点 |

**助忆技术**：
- **情境重构**：让受访者重温事件发生的步骤
  - 例：投票问题→先问"在哪里投票""是否请假""如何转车"
- **时间锚点**：使用日历标记参考期间
- **辅助问题序列**：先问一般情况，再问特定时间

### 2.4 减少社会期望偏差

**四类设计策略**：

1. **提高问题适当性感知**
   - 向受访者解释问题与研究目标的关联
   - 使用过滤问题，只对相关人群询问敏感问题

2. **处理答案含义**
   - 提供中性说明："两种答案都是可接受的"
   - 例："人们有时不去投票，是因为对选举不感兴趣、工作脱不开身..."

3. **调整报告详细程度**
   - 使用一般性问题建立基准
   - 例：先问"你通常喝几杯"，再问"昨天喝了几杯"

4. **代码作答**
   - 使用自填式问卷
   - CAPI（电脑辅助个人访谈）
   - 录音问题+答题纸

## 三、主观状态测量的问题设计

### 3.1 评价量表设计

**类别数量选择**：
| 类别数 | 适用场景 | 优缺点 |
|--------|----------|--------|
| 2-3类 | 简单分类 | 容易完成，信息量小 |
| 5-10类 | 大多数评价任务 | 信息量适中 |
| 10类以上 | 精细区分 | 边际效益递减 |

**量表类型**：
- **数字量表**：1-10分（需明确两端含义）
- **形容词量表**：极差→极好
- **同意-不同意量表**：需谨慎使用（认知复杂、维度混合）

**量表设计的陷阱**：
- ❌ 选项顺序影响答案（从"极差"到"极佳" vs 相反）
- ❌ 选项标签改变参考点（"好"的定义因量表而异）

### 3.2 观点问题设计

**常见缺陷**：多重维度混合

**案例分析**：
- ❌ "美国正远离真正的美国生活方式，可能有必要用武力来恢复它。"
  - 包含3个议题：距离多远？是否应恢复？是否用武力？
  
- ✅ 拆分为单一维度问题

**"同意-不同意"形式的问题**：
1. 明确界定问题措辞
2. 确保只呈现单一观点或问题
3. 避免认知复杂性（否定式表述）
4. 提供中间选项或"不知道"选项

### 3.3 知识测量方法

| 方法 | 优点 | 缺点 |
|------|------|------|
| 自报知识 | 简单 | 主观性强 |
| 对-错题 | 易评分 | 随机猜对率高 |
| 多项选择 | 易评分 | 依赖似是而非选项设计 |
| 开放式简答 | 无虚假正确 | 难评分 |

**似是而非选项设计原则**：
- 错误选项应与正确答案有相似性
- 避免明显错误的选项（如问第13任总统，选项放华盛顿、尼克松）

## 四、问卷设计通用原则

### 4.1 七项核心原则

```
原则1：只问与研究目标相关的问题
原则2：确保问题对所有受访者都适用
原则3：推敲措辞，确保每位受访者回答相同问题
原则4：选择所有受访者都理解的词语
原则5：清楚沟通答案类型
原则6：设计简洁易用的问卷格式
原则7：必要时培训受访者
```

### 4.2 问卷格式设计

**访员管理问卷要点**：
1. 可选择用词用括号标注
2. 访员指令与问题用词明确区分（加粗/阴影）
3. 跳答指令清晰一致
4. 答案记录方式统一

**自填式问卷要点**：
1. 跳答路径一目了然
2. 回答方式始终如一
3. 尽量减少开放式问题
4. 问题间距充足，易于阅读

### 4.3 常见错误与修正

| 错误类型 | 错误示例 | 修正示例 |
|----------|----------|----------|
| 答案形式不明确 | "你何时搬到堪萨斯城？" | "你在哪一年搬到堪萨斯城？" |
| 术语未界定 | "你经常锻炼吗？" | "你每周有几天进行至少20分钟的中等强度运动？" |
| 社会期望诱导 | "你有借书证吗？" | "许多人从图书馆借书，其他人则自己买书...请问你现在有或没有借书证？" |
| 时间框架模糊 | "你最近看过病吗？" | "在过去14天里，你看过几次病？" |

## 五、问题预调查评估

### 5.1 三种核心评估方法

```
┌────────────────────────────────────────────────────────────┐
│  方法1：专题小组讨论 (Focus Group)                         │
│  • 目的：探索受访者对问题的理解                            │
│  • 时机：问卷设计早期                                      │
│  • 输出：概念理解差异、术语歧义                            │
├────────────────────────────────────────────────────────────┤
│  方法2：深度个人访谈 (Cognitive Interview)                 │
│  • 目的：探查认知过程                                      │
│  • 技术：大声思考、释义、回忆探查                          │
│  • 输出：理解问题、信息检索、判断形成过程                  │
├────────────────────────────────────────────────────────────┤
│  方法3：田野预调查 (Field Pretest)                         │
│  • 目的：模拟真实调查情境                                  │
│  • 技术：行为编码、访员评估                                │
│  • 输出：实际执行问题、管理困难                            │
└────────────────────────────────────────────────────────────┘
```

### 5.2 行为编码技术

**编码类别**：
| 类别 | 含义 | 指示问题 |
|------|------|----------|
| 念读问题 | 访员是否逐字念出问题 | 措辞拗口、垂悬从句 |
| 正确跳答 | 跳答是否无误 | 跳答指令不清 |
| 打断 | 受访者是否打断问题 | 问题结构、选项呈现时机 |
| 重复问题 | 是否需要重复 | 理解困难 |
| 要求澄清 | 受访者要求解释 | 术语未界定、题意不清 |
| 不适当答案 | 答案不在选项范围内 | 答案类型未明确沟通 |

**评估标准**：
- 当≥15%的访谈出现某一行为时，表明问题可能存在可修补的缺陷

### 5.3 访员评估表

**三个评估维度**：
1. 逐字念出问题是否有困难？
2. 受访者是否有理解问题的困难？
3. 受访者是否有作答困难？

**评级标准**：
- A = 没有明显缺陷
- B = 可能有缺陷
- C = 肯定有缺陷

### 5.4 追问技术（Probes）

**理解追问示例**：
```
问题：上个月，你有几次去看病？

追问1：你的数字是否包括打电话咨询医生而非亲自看门诊？
追问2：上个月你是否在电话里与医生讨论过健康状况？
追问3：如果通过电话，你会把它算作答案吗？
追问4：你的数字是否包括看精神科医生？
```

**叙述性答案追问**：
- 让受访者解释答案的理由
- 例："当你说健康状况是'好'时，你出于什么考虑做出这个评价？"

## 六、问题效度评估

### 6.1 四种效度评估策略

| 策略 | 方法 | 目的 |
|------|------|------|
| 研究相关性 | 分析答案间可预测关系 | 验证测量有效性 |
| 记录核对 | 与官方记录比较 | 验证事实问题准确性 |
| 替代形式比较 | 用不同措辞问可比样本 | 识别措辞影响 |
| 再测一致性 | 两次询问相同问题 | 评估信度 |

### 6.2 替代形式比较案例

**研究发现**：
- "买不起所需物品的低收入者" vs "接受福利救济的人"
  - 后者引发的情绪反应更消极
  - 证明"福利"一词混淆了两个不同议题

**"不知道"选项的影响**：
- 明确提供"不知道"选项 → 选择率显著增加
- 不提供时，受访者倾向于坚持使用该答案

### 6.3 效度评估的局限

- 记录本身可能有误差
- 真实答案可能随时间变化
- 一致性不等于正确性

## 七、实用工具与模板

### 7.1 问题设计检查清单

```
□ 问题目标是否明确？
□ 关键术语是否已界定？
□ 所有受访者是否适用？
□ 受访者是否拥有回答所需信息？
□ 问题是否能在合理努力下回答？
□ 答案形式是否明确？
□ 是否减少了社会期望偏差？
□ 问题措辞是否清晰简洁？
□ 是否避免了双重问题？
□ 是否避免了否定式表述？
```

### 7.2 问题目标大纲模板

```
调查目的：[研究目标]

测量目标1：[目标描述]
- 子目标A：[具体信息需求]
- 子目标B：[具体信息需求]

测量目标2：[目标描述]
...

分析计划：
- 如何用信息实现研究目标
- 每个问题的分析角色
```

### 7.3 预调查报告模板

```
预调查总结报告

1. 问题表现数据
   - 行为编码结果
   - 访员评估汇总
   - 受访者反馈

2. 识别的问题缺陷
   - 理解不一致问题
   - 管理困难问题
   - 答案质量问题

3. 建议修改
   - 具体修改方案
   - 预期效果

4. 需要进一步测试的问题
```

## 八、关键洞察与最佳实践

### 8.1 核心洞察

1. **问题设计是科学而非艺术**
   - 有统一标准，不是"情人眼里出西施"
   - 最高标准：信度和效度

2. **评估是必要环节**
   - 预调查评估不是可选项
   - 作者最大希望：提高问题评估水平

3. **措辞改进效益 > 抽样改进**
   - 问题模糊产生的误差可达20%-30%
   - 改进措辞比扩大样本更具成本效益

4. **所有答案都有语境**
   - 量表选项改变参考点
   - 问题顺序影响答案
   - 主题框架影响答案含义

### 8.2 黄金法则

> **"一个好的问题是指这样的问题：它所引出的答案是对我们想要描述的东西可靠的、有效的测量。"**

> **"在调查中使用问题之前，进行问题评估是提高调查研究质量的最容易、最有成本效益的步骤。"**

## 九、应用场景

### 9.1 适用领域

- 社会科学研究调查
- 市场研究问卷
- 政府统计调查
- 医疗健康调查
- 教育评估问卷
- 员工满意度调查
- 客户体验调查

### 9.2 与相关方法论的关联

| 方法论 | 关系 |
|--------|------|
| 麦肯锡结构化战略思维 | 问卷设计是"定义问题"和"收集数据"的具体实现 |
| 七步问题解决法 | 问卷评估是"验证假设"的关键环节 |
| 第一性原理思维 | 从测量本质出发设计问题，而非模仿现有问卷 |
| 培训体系设计 | 问卷是培训需求分析的重要数据收集工具 |
